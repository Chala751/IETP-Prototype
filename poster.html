1 Introduction & Objectives

Conventional lighting wastes energy because rooms stay lit even when daylight is sufficient or no one is present. This project builds a low-cost IoT node that measures ambient light intensity in real time, streams it to a Next.js web dashboard over Wi-Fi, and triggers automatic LED control based on a user-defined threshold.

Specific Objectives

  • Measure ambient light using an LDR / BH1750 sensor
  • Automate LED control via ESP32 with threshold logic
  • Build a responsive Next.js dashboard for live visualization
  • Establish ESP32 ↔ web app communication using REST APIs

6 Firmware & Dashboard Logic

ESP32 Loop (Arduino · C++)

  • connectWiFi() — joins "Wokwi-GUEST" / local SSID at boot
  • fetchControl() — GET /api/light, parses threshold & ledMode
  • analogRead(GPIO34) → raw (0–4095) → map() to brightness %
  • Auto rule → LED ON when brightness < threshold; OFF otherwise
  • Override → ledMode = on / off forces LED state
  • postState() — POST raw, value, threshold, ledStatus, deviceId

10 Conclusion & Future Work
The IoT-Based Light Intensity Monitoring & Smart Control System validates an end-to-end pipeline from an ESP32 sensor node (LDR on GPIO34, LED on GPIO23) to a Next.js + MongoDB cloud dashboard over REST APIs. The Wokwi simulation achieved 1.2 s average API latency, a 98 % packet-success rate, a 300 ms LED response and ±2 % ADC stability, confirming the architecture is viable for small-scale smart-environment automation. Future work: physical hardware deployment with PCB & relay-controlled fixture, dual-threshold hysteresis to prevent flicker, TLS + JWT for secure communication, persistent storage via ESP32 NVS, MQTT alongside REST for low-latency scaling, mobile app, and ML-based occupancy / daylight prediction.
